# SDK Integration Guide

SDKs (Software Development Kits) are the bridge between your app and ad platforms. Without them, ad platforms cannot track who installed your app, what they do inside it, or whether your ads are working. This guide covers every major SDK you need to install, with code examples for React Native, iOS, and Android.

<Tip>
**Why SDKs matter**: When you run an ad campaign, the ad platform needs to know two things: (1) did the person who saw the ad actually install your app? and (2) did they do something valuable (subscribe, purchase, complete onboarding)? SDKs are what send this information from your app back to the ad platform. Without them, your campaigns are flying blind.
</Tip>

---

## Which SDKs Do You Need?

This depends on which ad platforms you use and whether you use an MMP (Mobile Measurement Partner):

| Scenario | SDKs to Install |
|---|---|
| **Meta ads only** | Meta SDK + ATT (iOS) |
| **Meta + Google** | Meta SDK + Firebase SDK + ATT |
| **Meta + Google + TikTok** | Meta SDK + Firebase SDK + TikTok SDK + ATT |
| **Using an MMP (Airbridge, AppsFlyer, etc.)** | MMP SDK + ATT (the MMP handles most platform integrations) |
| **Apple Search Ads** | Apple AdServices framework (no separate SDK needed) |

<Tip>
**Pro Tip**: If you plan to advertise on 3+ platforms, strongly consider using an **MMP** (like Airbridge, AppsFlyer, Adjust, or Branch) instead of installing each platform's SDK individually. An MMP provides a single SDK that handles attribution across all platforms, deduplicates installs, and gives you a unified dashboard. It simplifies your code and gives you better data. Airbridge is worth a look if you want a modern, developer-friendly SDK with strong documentation and unified web+app measurement built in.
</Tip>

## Meta SDK (Facebook/Instagram)

The Meta SDK is the most commonly used ad SDK. It tracks installs, in-app events, and enables deep linking from Meta ads.

<Tabs>
  <Tab title="React Native">
    **Installation**

    ```bash
    npm install react-native-fbsdk-next
    cd ios && pod install && cd ..
    ```

    **iOS Configuration** (`ios/YourApp/Info.plist`):

    ```xml
    <key>FacebookAppID</key>
    <string>YOUR_APP_ID</string>
    <key>FacebookClientToken</key>
    <string>YOUR_CLIENT_TOKEN</string>
    <key>FacebookDisplayName</key>
    <string>Your App Name</string>
    <key>CFBundleURLTypes</key>
    <array>
      <dict>
        <key>CFBundleURLSchemes</key>
        <array>
          <string>fbYOUR_APP_ID</string>
        </array>
      </dict>
    </array>
    ```

    **Android Configuration** (`android/app/src/main/res/values/strings.xml`):

    ```xml
    <string name="facebook_app_id">YOUR_APP_ID</string>
    <string name="facebook_client_token">YOUR_CLIENT_TOKEN</string>
    ```

    **Initialize and Track Events**:

    ```typescript
    import { Settings, AppEventsLogger } from 'react-native-fbsdk-next';

    // Initialize at app startup
    Settings.initializeSDK();

    // Standard events
    AppEventsLogger.logEvent('fb_mobile_complete_registration');
    AppEventsLogger.logPurchase(9.99, 'USD');
    AppEventsLogger.logEvent('fb_mobile_add_to_cart', {
      fb_content_id: 'product_123',
      fb_content_type: 'product',
      _valueToSum: 29.99,
      fb_currency: 'USD',
    });

    // Custom events
    AppEventsLogger.logEvent('tutorial_complete');
    AppEventsLogger.logEvent('level_complete', { level: 10 });
    ```
  </Tab>
  <Tab title="iOS (Swift)">
    **Installation** (CocoaPods):

    ```ruby
    pod 'FBSDKCoreKit'
    ```

    **Initialize** (`AppDelegate.swift`):

    ```swift
    import FBSDKCoreKit

    func application(
      _ application: UIApplication,
      didFinishLaunchingWithOptions launchOptions: [UIApplication.LaunchOptionsKey: Any]?
    ) -> Bool {
      ApplicationDelegate.shared.application(
        application,
        didFinishLaunchingWithOptions: launchOptions
      )
      return true
    }
    ```

    **Track Events**:

    ```swift
    import FBSDKCoreKit

    // Standard events
    AppEvents.shared.logEvent(.completedRegistration)
    AppEvents.shared.logPurchase(amount: 9.99, currency: "USD")
    AppEvents.shared.logEvent(.addedToCart, parameters: [
      .contentID: "product_123",
      .contentType: "product",
      .currency: "USD",
    ])

    // Custom events
    AppEvents.shared.logEvent(AppEvents.Name("tutorial_complete"))
    ```
  </Tab>
  <Tab title="Android (Kotlin)">
    **Installation** (`build.gradle`):

    ```kotlin
    implementation 'com.facebook.android:facebook-android-sdk:latest.release'
    ```

    **Initialize** (`Application` class):

    ```kotlin
    import com.facebook.FacebookSdk

    class MyApplication : Application() {
      override fun onCreate() {
        super.onCreate()
        FacebookSdk.sdkInitialize(applicationContext)
      }
    }
    ```

    **Track Events**:

    ```kotlin
    import com.facebook.appevents.AppEventsLogger

    val logger = AppEventsLogger.newLogger(this)

    // Standard events
    logger.logEvent("fb_mobile_complete_registration")
    logger.logPurchase(BigDecimal(9.99), Currency.getInstance("USD"))

    // Custom events
    logger.logEvent("tutorial_complete")
    logger.logEvent("level_complete", Bundle().apply {
      putInt("level", 10)
    })
    ```
  </Tab>
</Tabs>

## Firebase / Google Analytics SDK

Firebase provides analytics, crash reporting, and integration with Google Ads. It is the primary SDK for Google ad campaigns.

<Tabs>
  <Tab title="React Native">
    **Installation**:

    ```bash
    npm install @react-native-firebase/app @react-native-firebase/analytics
    cd ios && pod install && cd ..
    ```

    **Track Events**:

    ```typescript
    import analytics from '@react-native-firebase/analytics';

    // Standard events
    await analytics().logSignUp({ method: 'email' });
    await analytics().logPurchase({
      value: 9.99,
      currency: 'USD',
      items: [{ item_id: 'premium_plan', item_name: 'Premium Monthly' }],
    });
    await analytics().logTutorialComplete();

    // Custom events
    await analytics().logEvent('level_complete', { level: 10 });

    // Set user properties
    await analytics().setUserProperty('subscription_tier', 'premium');
    ```
  </Tab>
  <Tab title="iOS (Swift)">
    **Installation** (CocoaPods):

    ```ruby
    pod 'FirebaseAnalytics'
    ```

    **Initialize** (`AppDelegate.swift`):

    ```swift
    import FirebaseCore

    func application(
      _ application: UIApplication,
      didFinishLaunchingWithOptions launchOptions: [UIApplication.LaunchOptionsKey: Any]?
    ) -> Bool {
      FirebaseApp.configure()
      return true
    }
    ```

    **Track Events**:

    ```swift
    import FirebaseAnalytics

    Analytics.logEvent(AnalyticsEventSignUp, parameters: [
      AnalyticsParameterMethod: "email"
    ])

    Analytics.logEvent(AnalyticsEventPurchase, parameters: [
      AnalyticsParameterValue: 9.99,
      AnalyticsParameterCurrency: "USD",
    ])

    // Custom event
    Analytics.logEvent("level_complete", parameters: ["level": 10])
    ```
  </Tab>
  <Tab title="Android (Kotlin)">
    **Add to `build.gradle`** (app level):

    ```kotlin
    implementation platform('com.google.firebase:firebase-bom:32.0.0')
    implementation 'com.google.firebase:firebase-analytics-ktx'
    ```

    **Track Events**:

    ```kotlin
    import com.google.firebase.analytics.FirebaseAnalytics
    import com.google.firebase.analytics.ktx.analytics
    import com.google.firebase.ktx.Firebase

    val firebaseAnalytics = Firebase.analytics

    // Standard events
    firebaseAnalytics.logEvent(FirebaseAnalytics.Event.SIGN_UP) {
      param(FirebaseAnalytics.Param.METHOD, "email")
    }

    firebaseAnalytics.logEvent(FirebaseAnalytics.Event.PURCHASE) {
      param(FirebaseAnalytics.Param.VALUE, 9.99)
      param(FirebaseAnalytics.Param.CURRENCY, "USD")
    }

    // Custom event
    firebaseAnalytics.logEvent("level_complete") {
      param("level", 10)
    }
    ```
  </Tab>
</Tabs>

## TikTok Business SDK

Required for TikTok ad campaigns. Tracks installs and in-app events.

<Tabs>
  <Tab title="React Native">
    ```bash
    npm install tiktok-business-sdk-react-native
    ```

    ```typescript
    import TikTokBusiness from 'tiktok-business-sdk-react-native';

    // Initialize
    TikTokBusiness.initializeSdk({
      appId: 'YOUR_APP_ID',
      tiktokAppId: 'YOUR_TIKTOK_APP_ID',
      debug: __DEV__,
    });

    // Track events
    TikTokBusiness.trackEvent('CompleteRegistration');
    TikTokBusiness.trackEvent('Purchase', {
      content_id: 'premium_plan',
      content_type: 'product',
      currency: 'USD',
      value: 9.99,
    });
    ```
  </Tab>
  <Tab title="iOS (Swift)">
    ```ruby
    # Podfile
    pod 'TikTokBusinessSDK'
    ```

    ```swift
    import TikTokBusinessSDK

    // Initialize in AppDelegate
    let config = TikTokConfig(appId: "YOUR_APP_ID", tiktokAppId: "YOUR_TIKTOK_APP_ID")
    TikTokBusiness.initializeSdk(config)

    // Track events
    TikTokBusiness.trackEvent("Purchase", withProperties: [
      "content_id": "premium_plan",
      "currency": "USD",
      "value": 9.99
    ])
    ```
  </Tab>
  <Tab title="Android (Kotlin)">
    ```kotlin

    TikTokBusinessSdk.trackEvent("Purchase", JSONObject().apply {
    ```

## Apple Search Ads Attribution

Apple Search Ads does not require a separate SDK. It uses the built-in AdServices framework (iOS 14.3+).

```swift
import AdServices

func fetchAppleSearchAdsAttribution() {
    if #available(iOS 14.3, *) {
        do {
            let token = try AAAttribution.attributionToken()
            sendTokenToServer(token)
        } catch {
            print("Attribution token unavailable: \(error)")
        }
    }
}

func sendTokenToServer(_ token: String) {
    guard let url = URL(string: "https://api-adservices.apple.com/api/v1/") else { return }

    var request = URLRequest(url: url)
    request.httpMethod = "POST"
    request.setValue("text/plain", forHTTPHeaderField: "Content-Type")
    request.httpBody = Data(token.utf8)

    URLSession.shared.dataTask(with: request) { data, _, error in
        guard let data = data else { return }
        // Parse attribution JSON response
        if let json = try? JSONSerialization.jsonObject(with: data) as? [String: Any] {
            print("Attribution data: \(json)")
        }
    }.resume()
}
```

<Tip>
**Pro Tip**: Call the attribution API on **first launch only**. Store the result and do not call it again. If you use an MMP, it handles Apple Search Ads attribution automatically — you do not need to write this code.
</Tip>

## MMP Integration

If you advertise on multiple platforms, an MMP provides a single SDK that handles all attribution. Below are integration examples for two popular MMPs.

<Tabs>
  <Tab title="React Native (Airbridge)">
    ```bash
    npm install airbridge-react-native-sdk
    ```

    ```typescript
    import Airbridge from 'airbridge-react-native-sdk';

    // Initialize at app startup
    Airbridge.init({
      appName: 'YOUR_APP_NAME',
      appToken: 'YOUR_APP_TOKEN',
    });

    // Track events (works across all connected platforms)
    Airbridge.trackEvent('purchase', {
      value: 9.99,
      attributes: {
        currency: 'USD',
        productId: 'premium_plan',
      },
    });

    Airbridge.trackEvent('sign_up', {
      attributes: {
        method: 'email',
      },
    });
    ```
  </Tab>
  <Tab title="React Native (AppsFlyer)">
    ```bash
    npm install react-native-appsflyer
    ```

    ```typescript
    import appsFlyer from 'react-native-appsflyer';

    // Initialize
    appsFlyer.initSdk({
      devKey: 'YOUR_DEV_KEY',
      isDebug: __DEV__,
      appId: 'YOUR_IOS_APP_ID', // iOS only
      onInstallConversionDataListener: true,
    });

    // Track events (works across all connected platforms)
    appsFlyer.logEvent('af_purchase', {
      af_revenue: 9.99,
      af_currency: 'USD',
      af_content_id: 'premium_plan',
    });

    appsFlyer.logEvent('af_complete_registration', {
      af_registration_method: 'email',
    });
    ```
  </Tab>
  <Tab title="iOS (Swift — Airbridge)">
    ```swift
    import AirBridge

    // Initialize in AppDelegate
    AirBridge.setting.appName = "YOUR_APP_NAME"
    AirBridge.setting.appToken = "YOUR_APP_TOKEN"
    AirBridge.getInstance("YOUR_APP_TOKEN", appName: "YOUR_APP_NAME")

    // Track events
    AirBridge.trackEvent(category: "purchase", attributes: [
      "value": 9.99,
      "currency": "USD",
      "productId": "premium_plan"
    ])
    ```
  </Tab>
  <Tab title="iOS (Swift — AppsFlyer)">
    ```swift
    import AppsFlyerLib

    // Initialize in AppDelegate
    AppsFlyerLib.shared().appsFlyerDevKey = "YOUR_DEV_KEY"
    AppsFlyerLib.shared().appleAppID = "YOUR_APP_ID"
    AppsFlyerLib.shared().isDebug = true // Remove in production
    AppsFlyerLib.shared().start()

    // Track events
    AppsFlyerLib.shared().logEvent("af_purchase", withValues: [
      "af_revenue": 9.99,
      "af_currency": "USD",
      "af_content_id": "premium_plan"
    ])
    ```
  </Tab>
</Tabs>

<Tip>
**Pro Tip**: Airbridge's SDK supports both web and app tracking with a single integration, which means you can measure the full user journey from your landing page through to in-app conversions. This is particularly useful if you run campaigns that send users to a website first before directing them to the app store.
</Tip>

## iOS App Tracking Transparency (ATT)

ATT is required for iOS 14.5+. You must ask permission before tracking users across apps and websites.

<Tabs>
  <Tab title="React Native">
    ```bash
    npm install react-native-tracking-transparency
    ```

    ```typescript
    import {
      requestTrackingPermission,
      getTrackingStatus,
    } from 'react-native-tracking-transparency';
    import { Settings } from 'react-native-fbsdk-next';

    async function setupTracking() {
      const status = await requestTrackingPermission();

      if (status === 'authorized') {
        // User opted in - enable full tracking
        Settings.setAdvertiserTrackingEnabled(true);
      } else {
        // User opted out - limited tracking only
        Settings.setAdvertiserTrackingEnabled(false);
      }
    }

    // Call this early in app startup, but after a brief delay
    // (showing the prompt immediately on first launch has lower opt-in rates)
    ```

    Add to `ios/YourApp/Info.plist`:
    ```xml
    <key>NSUserTrackingUsageDescription</key>
    <string>This helps us show you relevant ads and improve your experience.</string>
    ```
  </Tab>
  <Tab title="iOS (Swift)">
    ```swift
    import AppTrackingTransparency
    import AdSupport

    func requestTrackingPermission() {
        if #available(iOS 14, *) {
            ATTrackingManager.requestTrackingAuthorization { status in
                switch status {
                case .authorized:
                    let idfa = ASIdentifierManager.shared().advertisingIdentifier
                    print("IDFA: \(idfa)")
                case .denied, .restricted:
                    print("Tracking denied")
                case .notDetermined:
                    print("Not determined")
                @unknown default:
                    break
                }
            }
        }
    }
    ```
  </Tab>
</Tabs>

<Tip>
**Pro Tip**: When to show the ATT prompt matters. Showing it immediately on first launch gives lower opt-in rates (15-20%). Showing it after the user has experienced value from your app (after completing onboarding, after using a key feature) gives higher opt-in rates (25-35%). However, do not wait too long — tracking works best when enabled from the start.
</Tip>

## Standard Events Reference

Use these standard event names across platforms for consistency:

| User Action | Meta Event | Firebase Event | TikTok Event | AppsFlyer Event |
|---|---|---|---|---|
| Registration | `fb_mobile_complete_registration` | `sign_up` | `CompleteRegistration` | `af_complete_registration` |
| Purchase | `fb_mobile_purchase` | `purchase` | `Purchase` | `af_purchase` |
| Add to Cart | `fb_mobile_add_to_cart` | `add_to_cart` | `AddToCart` | `af_add_to_cart` |
| Start Trial | `StartTrial` | `begin_checkout` | `StartTrial` | `af_start_trial` |
| Tutorial Complete | `fb_mobile_tutorial_completion` | `tutorial_complete` | `CompleteTutorial` | `af_tutorial_completion` |
| Level Complete | Custom | Custom | Custom | Custom |
| Subscribe | `Subscribe` | `purchase` | `Subscribe` | `af_subscribe` |

## Testing and Debugging

### How to Verify Your SDK Integration

<Steps>
  <Step title="Enable debug mode">
    Each SDK has a debug mode. Enable it during development to see events in the console:
    - **Meta**: Use the Events Manager "Test Events" tool
    - **Firebase**: Events appear in DebugView (enable with `-FIRAnalyticsDebugEnabled` launch argument)
    - **TikTok**: Set log level to debug in SDK config
    - **Airbridge**: Use the Airbridge dashboard's real-time event viewer
    - **AppsFlyer**: Set `isDebug = true`
  </Step>
  <Step title="Trigger events on a real device">
    Simulators/emulators do not always send events correctly. Always test on a real device for final verification.
  </Step>
  <Step title="Check the platform dashboard">
    After triggering events, verify they appear in each platform's dashboard:
    - **Meta**: Events Manager > Test Events
    - **Firebase**: Analytics > DebugView (real-time) or Events (delayed 24-48 hours)
    - **TikTok**: Events Manager > App Events
    - **Airbridge**: Dashboard > Real-time Logs
    - **AppsFlyer**: Dashboard > Activity Log
  </Step>
  <Step title="Test the full flow">
    1. Delete the app from your device
    2. Click a test ad link (each platform provides test URLs)
    3. Install the app
    4. Open the app and trigger key events
    5. Verify the install and events appear in the platform dashboard
  </Step>
</Steps>

### Common Issues

| Issue | Likely Cause | Fix |
|---|---|---|
| Events not appearing | SDK not initialized before events fire | Move initialization to app startup |
| Install not attributed | Test device has existing data | Reset advertising ID and clear app data |
| Events delayed | Normal — some platforms batch events | Wait 15-30 minutes, check again |
| Duplicate events | Event fired multiple times in code | Add guards to prevent re-firing |
| Wrong revenue values | Currency or value format incorrect | Ensure numeric value and ISO currency code |

## Integration Checklist

Before launching ad campaigns, verify:

| Item | Platform | Status |
|---|---|---|
| SDK installed and initialized | All platforms you use | Required |
| Install tracking working | All platforms | Required |
| Key events tracked (registration, purchase) | All platforms | Required |
| ATT prompt implemented | iOS only | Required |
| Revenue tracking with correct currency | All platforms | Strongly recommended |
| Debug mode disabled for production | All platforms | Required before app store submission |
| Events verified in platform dashboard | All platforms | Required |

## Key Takeaways

1. **SDKs are required** for ad platform optimization — without them, your campaigns cannot learn who to target
2. **Consider an MMP** if you use 3+ platforms — one SDK instead of many
3. **ATT is mandatory** on iOS — implement it early and test the opt-in rate
4. **Track the events that matter** — purchases and registrations are more valuable than page views
5. **Test on real devices** — simulators do not always send events correctly
6. **Standard event names** vary by platform — use the reference table above for consistency

<Tip>
**Final thought**: SDK integration is a one-time investment that pays dividends for as long as you run ads. Take the time to get it right, verify everything is working, and then you can focus entirely on creative, targeting, and strategy. A properly instrumented app is the foundation of effective advertising.
</Tip>
